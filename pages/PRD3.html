1. Executive Summary

This Product Requirements Document (PRD) defines the specifications for developing an integrated Jobs-to-be-Done (JTBD) driven campaign management and attribution system. The system will enable marketing teams to create, manage, and optimize campaigns based on JTBD principles while providing comprehensive multi-touch attribution with machine learning capabilities.

The solution addresses key challenges faced by marketing and sales teams:

  • Disconnect between marketing content and customer job needs
  • Ineffective campaign targeting due to poor job alignment
  • Inability to accurately attribute conversions across multiple touchpoints
  • Lack of feedback loops between sales outcomes and marketing content

This system will integrate with existing website infrastructure, CRM systems (primarily Odoo), and content management processes to create a comprehensive JTBD-driven marketing ecosystem.

Update Notes (v1.2): This document has been fully reconciled with the system architecture and customer journey flows based on our comprehensive validation process. Key enhancements include standardized naming conventions, clarified component relationships, expanded integration specifications, comprehensive privacy controls, visual tools framework, and complete documentation standards.

  1. Outbound Foundation

    • Target identification and enrichment

  5. Expansion Engine

    • Opportunity identification
    • Partner network integration
    • Success benchmarking
    • White space opportunity mapping
    • Portfolio evolution tracking

2.4.2 Journey Entry Points

The system supports multiple journey entry points with pathway-specific processes:

  1. Outbound Pathway

    • Entered through prospecting with jtbd_automation_source='outbound'
    • Emphasizes target identification and enrichment
    • Leverages market signals and trigger detection
    • Focused on proactive outreach based on signal strength

  2. Inbound Pathway

    • Entered through prospecting with jtbd_automation_source='inbound'
    • Emphasizes job validation and outcome definition
    • Leverages collaborative workshops and adaptive forms
    • Focused on response to expressed interest

  3. Expansion Pathway

    • Entered through delivery process completion
    • Emphasizes white space analysis and portfolio evolution
    • Leverages success benchmarking and implementation learnings
    • Focused on extending value through related jobs

for record in self:
if not record.jtbd_job_statement:
record.jtbd_job_clarity_score = 0
continue

    \# Base score starting at 50  
    score \= 50  
      
    \# Check for situation component  
    if "when " in record.jtbd\_job\_statement.lower():  
        score \+= 15  
          
    \# Check for motivation component      
    if " i want to " in record.jtbd\_job\_statement.lower():  
        score \+= 15  
          
    \# Check for outcome component  
    if " so i can " in record.jtbd\_job\_statement.lower():  
        score \+= 15  
          
    \# Adjust for statement length (penalize if too short)  
    if len(record.jtbd\_job\_statement) \< 30:  
        score \-= 25

  • jtbd_momentum_score: Calculation of decision momentum

@api.depends(‘push_score’, ‘pull_score’, ‘anxiety_score’, ‘habit_score’)
def _compute_momentum_score(self):
for record in self:
# Combined forces pushing toward change
change_forces = record.push_score + record.pull_score

    \# Combined forces maintaining status quo  
    inertia\_forces \= record.anxiety\_score \+ record.habit\_score  
      
    \# If no forces, default to middle  
    if change\_forces \== 0 and inertia\_forces \== 0:  
        record.momentum\_score \= 50  
        continue  
          
    \# Calculate momentum ratio and convert to 0-100 scale  
    if inertia\_forces \> 0:  
        momentum\_ratio \= change\_forces / (change\_forces \+ inertia\_forces)  
        record.momentum\_score \= int(momentum\_ratio \* 100\)  
    else:  
        record.momentum\_score \= min(change\_forces \* 2, 100\)  \# Double the force if no inertia

@api.depends(‘momentum_score’, ‘trigger_window’, ‘contract_loss’)
def _compute_signal_strength(self):

# Start with momentum score
signal = record.momentum_score

    \# Adjust based on trigger window
