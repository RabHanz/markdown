JTBD-Driven Campaign Management & Attribution System

Product Requirements Document

Version 1.1 | April 16, 2025

1. Executive Summary

This Product Requirements Document (PRD) defines the specifications for developing an integrated Jobs-to-be-Done (JTBD) driven campaign management and attribution system. The system will enable marketing teams to create, manage, and optimize campaigns based on JTBD principles while providing comprehensive multi-touch attribution with machine learning capabilities.

The solution addresses key challenges faced by marketing and sales teams:

  • Disconnect between marketing content and customer job needs
  • Ineffective campaign targeting due to poor job alignment
  • Inability to accurately attribute conversions across multiple touchpoints
  • Lack of feedback loops between sales outcomes and marketing content

This system will integrate with existing website infrastructure, CRM systems (primarily Odoo), and content management processes to create a comprehensive JTBD-driven marketing ecosystem.

Update Notes (v1.1): This document has been updated based on the comprehensive validation process conducted across nine critical documents, which identified significant gaps and enhancement opportunities in the original system design. Key enhancements include expanded force analysis framework, visual tools implementation, comprehensive data structure specification, process automation framework, and AI integration capabilities.

2. System Architecture

The system consists of eight primary components that work together to create a complete JTBD-driven campaign management and attribution ecosystem:

  1. JTBD Radar (Data Collection & Analysis)
  2. Force Analysis Engine
  3. Content Brain
  4. Guided Wizards
  5. Personalization Engine
  6. Component Library
  7. Conversion Layer
  8. Analytics & Optimization

Each component addresses specific aspects of the campaign management and attribution process while maintaining JTBD principles throughout.

@api.depends('jtbd_job_statement')
def _compute_job_clarity_score(self):
    for record in self:
        if not record.jtbd_job_statement:
            record.jtbd_job_clarity_score = 0
            continue
            
        # Base score starting at 50
        score = 50
        
        # Check for situation component
        if "when " in record.jtbd_job_statement.lower():
            score += 15
            
        # Check for motivation component    
        if " i want to " in record.jtbd_job_statement.lower():
            score += 15
            
        # Check for outcome component
        if " so i can " in record.jtbd_job_statement.lower():
            score += 15
            
        # Adjust for statement length (penalize if too short)
        if len(record.jtbd_job_statement) < 30:
            score -= 25
        
        # Cap at 0-100 range
        record.jtbd_job_clarity_score = max(0, min(100, score))
  • jtbd_momentum_score: Calculation of decision momentum
@api.depends('push_score', 'pull_score', 'anxiety_score', 'habit_score')
def _compute_momentum_score(self):
    for record in self:
        # Combined forces pushing toward change
        change_forces = record.push_score + record.pull_score
        
        # Combined forces maintaining status quo
        inertia_forces = record.anxiety_score + record.habit_score
        
        # If no forces, default to middle
        if change_forces == 0 and inertia_forces == 0:
            record.momentum_score = 50
            continue
            
        # Calculate momentum ratio and convert to 0-100 scale
        if inertia_forces > 0:
            momentum_ratio = change_forces / (change_forces + inertia_forces)
            record.momentum_score = int(momentum_ratio * 100)
        else:
            record.momentum_score = min(change_forces * 2, 100)  # Double the force if no inertia
  • jtbd_signal_strength: Combined measure of intent and urgency
@api.depends('momentum_score', 'trigger_window', 'contract_loss')
def _compute_signal_strength(self):
    for record in self:
        # Start with momentum score
        signal = record.momentum_score
        
        # Adjust based on trigger window
        window_adjustments = {
            'immediate': 30,
            'short': 15,
            'medium': 0,
            'long': -15,
            False: 0
        }
        signal += window_adjustments.get(record.trigger_window, 0)
        
        # Adjust based on contract loss
        if record.contract_loss:
            signal += 10
            
        # Ensure signal is within 0-100 range
        record.signal_strength = max(min(signal, 100), 0)
